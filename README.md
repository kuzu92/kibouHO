# Role Assigner (同率順位対応・2つのモードを選べる役職自動割り当てツール)

メンバーから提出された希望順位をもとに、各役職の定員に合わせて自動で均等に割り振るPythonツールです。GitHub Actionsに対応しており、PCにPython環境がない人でもブラウザ上で完結して実行できます。

## ✨ 特徴
- **同率順位に対応**: 「A役もB役もどちらも1位」といった柔軟な希望提出が可能です。
- **選べる2つの割り当てロジック**: 組織の目的に応じて設定ファイルからモードを切り替えられます。
- **公平な抽選**: 同一条件で競合が発生した場合は、プログラムが自動でランダムに抽選します。

---

## 🛠 2つの割り当てモード

`config.yaml` の `mode` を書き換えることで、割り当ての基本方針を変更できます。

### 1. 第1希望最優先モード (`satisfaction_first`)
- **方針**: とにかく「第1希望」が通る人の数を最大化します。
- **向いているケース**: 強いこだわりを持つメンバーが多く、上位希望を叶えることが全体の満足度につながる場合。
- **注意点**: 人気の役職から漏れた人が、第3・第4希望などの下位希望に回されるリスクがあります。

### 2. ワースト回避モード (`fairness_first`)
- **方針**: 「第4希望（または希望外）」になってしまう不満度の高い人を最小限に抑えます。
- **向いているケース**: 「誰も大ハズレを引かせたくない」「全員に平均以上の満足度を与えたい」という公平性を重視する場合。
- **仕組み**: 下位希望になるほどペナルティ（不満度スコア）を重く計算し、組織全体の総不満度が最も低くなる組み合わせを自動選出します。

## 🚀 使い方 (GitHub上での実行手順)

利用者は、ご自身のパソコンにPythonをインストールする必要はありません。

### 1. 設定の編集
1. GitHub上で `config.yaml` を開きます。
2. 右上の鉛筆マーク（編集ボタン）をクリックします。
3. 動作モード（`mode`）、役職の定員（`roles`）、メンバーの希望順位（`preferences`）を書き換えます。
   - 数値が小さいほど優先度が高くなります（1 = 第一希望）。
   - 同じ数値を書くと同率順位になります。
4. 画面右上の **「Commit changes...」** ボタンを押して保存します。

### 2. ツールの実行
1. リポジトリ上部の **「Actions」** タブをクリックします。
2. 左側のメニューから **「Execute Role Assignment」** を選択します。
3. 右側に表示される **「Run workflow」** ボタンをクリックします。

### 3. 結果の確認
1. 実行完了後（数十秒）、緑色のチェックマークがついた履歴をクリックします。
2. 画面下部の **「Artifacts」** にある `assignment-result` から、結果が記載されたテキストファイルをダウンロードできます。
3. または、ログ内の「Run assignment script」の項目を開くことでも、画面上で直接結果を確認できます。

---

## 💻 ローカル環境（PC）での実行手順

手元のPCで直接動かしたい場合は以下の通りです。

### 必要なライブラリのインストール
```bash
pip install pyyaml
```

### 実行コマンド
```bash
python assign.py
```
実行すると、ターミナル上に割り当て結果の表が出力されます。
